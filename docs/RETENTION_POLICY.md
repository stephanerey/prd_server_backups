# Retention Policy

The default policy generated by `server-backup setup` is:

- `RETENTION_DAILY=14`
- `RETENTION_WEEKLY=8`
- `RETENTION_MONTHLY=12`

This policy is applied per target repository.

## Commands

Review what would be removed:

```bash
sudo server-backup repo prune <target> --dry-run
```

Apply the policy for real:

```bash
sudo server-backup repo prune <target> --yes
```

## Operational Guidance

- always run a dry-run before the first real prune
- real prune is destructive
- with only one recent snapshot, prune often removes nothing
- prune uses restic retention, not storage-provider lifecycle rules

## Meaning

- `daily`: keep the most recent daily snapshots
- `weekly`: keep the most recent weekly snapshots
- `monthly`: keep the most recent monthly snapshots

At least one of the three values must be greater than zero.

## Storage Impact

Higher values keep more history and use more space over time.
Lower values reduce storage growth but shorten recovery history.

Restic retention works inside the encrypted repository. It is independent from:

- NAS filesystem quotas
- object storage lifecycle rules
- provider-side retention or deletion policies
